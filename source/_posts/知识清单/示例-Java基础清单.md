## 🎯 知识点详细清单

### 1. 面向对象基础

#### 1.1 面向对象三大特性
- [ ] **封装**：理解访问修饰符、getter/setter的意义
- [ ] **继承**：继承的使用场景、方法重写、super关键字
- [ ] **多态**：编译时多态(重载)、运行时多态(重写)、动态绑定

**相关资源：**
- 📖 书籍：《Java核心技术 卷I》
- 🔗 参考链接：
- 📝 面试题：多态的实现原理？

---

#### 1.2 重写(Override)与重载(Overload)
- [ ] **重载规则**：方法名相同，参数列表不同
- [ ] **重写规则**：方法签名相同，返回类型兼容，访问权限不能更小
- [ ] **区别对比**：发生时期、参数要求、返回值要求、异常要求

**易错点：**
- 重载与重写的返回值要求区别
- private方法不能被重写
- static方法不能被重写，但可以被隐藏

---

#### 1.3 抽象类与接口
- [ ] **抽象类**：可以有成员变量、构造方法、普通方法
- [ ] **接口**：JDK 8前后变化(default、static方法)
- [ ] **对比选择**：什么时候用抽象类，什么时候用接口

**面试高频：**
> 抽象类和接口的区别？Java 8之后接口有什么变化？

---

### 2. Object类与常用类

#### 2.1 == 与 equals()
- [ ] **==**：比较基本类型值，引用类型地址
- [ ] **equals()**：默认比较地址，重写后比较内容
- [ ] **String的equals()**：已经重写，比较字符串内容

---

#### 2.2 hashCode() 与 equals()
- [ ] **契约关系**：equal的对象必须有相同的hashCode
- [ ] **HashMap中的应用**：为什么需要同时重写
- [ ] **重写最佳实践**：使用Objects.hash()或手动计算

**面试高频：**
> 为什么重写equals()必须重写hashCode()？
> 两个对象hashCode相同一定equals吗？

---

#### 2.3 String、StringBuffer、StringBuilder
- [ ] **String**：不可变性、常量池、intern()方法
- [ ] **StringBuilder**：可变、非线程安全、单线程首选
- [ ] **StringBuffer**：可变、线程安全(synchronized)、多线程使用
- [ ] **性能对比**：执行速度、内存占用

**深入理解：**
- String不可变的好处
- 字符串拼接的优化
- +号拼接的编译期优化

---

### 3. 集合框架

#### 3.1 ArrayList
- [ ] **底层实现**：Object数组
- [ ] **扩容机制**：1.5倍扩容，复制数组
- [ ] **线程安全问题**：多线程下的异常
- [ ] **fast-fail机制**：ConcurrentModificationException

**源码关键点：**
```java
// 扩容核心代码
grow(minCapacity);
Arrays.copyOf(elementData, newCapacity);
```

---

#### 3.2 HashMap
- [ ] **JDK 1.7**：数组+链表，头插法，扩容死循环问题
- [ ] **JDK 1.8**：数组+链表/红黑树，尾插法
- [ ] **扩容机制**：2倍扩容，rehash
- [ ] **put流程**：hash计算、冲突处理、链表转红黑树
- [ ] **线程安全问题**：死循环、数据丢失

**面试高频：**
> HashMap的底层原理？JDK 1.7和1.8有什么区别？
> 为什么用红黑树？链表转红黑树的阈值为什么是8？

---

#### 3.3 ConcurrentHashMap
- [ ] **JDK 1.7**：Segment分段锁
- [ ] **JDK 1.8**：CAS + synchronized，细粒度锁
- [ ] **size()实现**：CounterCell、baseCount
- [ ] **扩容机制**：多线程协助扩容
