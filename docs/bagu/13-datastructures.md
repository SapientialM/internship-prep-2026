## 1. 常见排序算法

| 算法 | 最好 | 平均 | 最差 | 空间 | 稳定 |
|:---|:---|:---|:---|:---|:---:|
| 快速排序 | O(n log n) | O(n log n) | O(n²) | O(log n) | ❌ |
| 归并排序 | O(n log n) | O(n log n) | O(n log n) | O(n) | ✅ |
| 堆排序 | O(n log n) | O(n log n) | O(n log n) | O(1) | ❌ |
| 冒泡排序 | O(n) | O(n²) | O(n²) | O(1) | ✅ |
| 插入排序 | O(n) | O(n²) | O(n²) | O(1) | ✅ |
| 计数排序 | O(n+k) | O(n+k) | O(n+k) | O(n+k) | ✅ |

**Java 中的 Arrays.sort()：**
- 基本类型用**双轴快排**（Dual-Pivot QuickSort）
- 对象类型用**TimSort**（归并 + 插入的混合排序）

## 2. 常见数据结构

### 树

**二叉搜索树（BST）：** 左 < 根 < 右，但可能退化为链表。

**平衡二叉树（AVL）：** 左右子树高度差 ≤ 1，通过旋转维持平衡，查找快但插入删除旋转开销大。

**红黑树：** 通过颜色约束（根黑、无连续红、黑高一致）维持近似平衡。插入删除旋转次数少（最多3次），JDK 的 TreeMap/HashMap(树化)、ConcurrentHashMap 都使用红黑树。

**B/B+树：** 多路搜索树，每个节点存储多个键值，减少树高度和磁盘 IO。B+树所有数据在叶子层，且有链表串联，适合范围查询。MySQL InnoDB 使用 B+树。

### 堆

**大顶堆 / 小顶堆** 用于 Top K 问题、优先级队列。JDK `PriorityQueue` 基于小顶堆实现。

### 跳表（SkipList）

多级索引的链表，期望复杂度 O(log n)，Redis ZSet 使用跳表实现。

## 3. 常见算法思想

| 思想 | 核心思路 | 典型问题 |
|:---|:---|:---|
| **双指针** | 快慢指针/左右指针 | 链表找环、有序数组 two sum |
| **滑动窗口** | 维护一个可变的窗口 | 最长无重复子串 |
| **二分查找** | 每次排除一半 | 旋转排序数组查找 |
| **动态规划** | 记忆化 + 状态转移 | 背包、最长公共子序列 |
| **回溯** | 递归尝试所有可能 + 剪枝 | 全排列、N皇后、组合总和 |
| **贪心** | 每步做最优选择 | 区间调度、跳跃游戏 |
| **分治** | 分解→解决→合并 | 归并排序 |

## 5. 算法刷题要点

### 链表
- 虚拟头节点（Dummy Node）
- 快慢指针（找中点、判环）
- 反转链表（迭代 + 递归）

### 二叉树
- 前中后序遍历（递归 + 迭代 + Morris）
- 层序遍历（BFS + Queue）
- 最近公共祖先 LCA

### 动态规划
- 解题框架：状态定义 → 转移方程 → 初始值 → 遍历顺序 → 输出
- 背包九讲：0-1背包、完全背包、多重背包
- 子序列 DP：最长递增子序列、最长公共子序列

### 回溯
- `void backtrack(路径, 选择列表)` 模板
- 排列（讲究顺序）、组合（不讲究顺序）、子集
