## ArrayList vs LinkedList：什么时候用哪个？

这是一个经典的面试问题，但很多答案只说了一半。

### 性能对比

| 操作 | ArrayList | LinkedList |
|:---|:---|:---|
| 尾部追加 | O(1) 均摊（快） | O(1) |
| 头部插入 | O(n) — 需要整体移动 | O(1) |
| 中间插入 | O(n) — 移动后半部分 | O(n) — 先遍历到位置 |
| 随机访问 | O(1) — 直接索引 | O(n) — 遍历 |
| 内存占用 | 连续内存，单元素空间小 | 每个元素需要额外存储前后指针 |
| CPU 缓存利用率 | 高（连续内存，缓存友好） | 低（节点分散在堆中，缓存未命中多） |

### 实际建议

1. **绝大多数场景用 ArrayList** — 即使有插入需求，现代 CPU 的数组拷贝（`System.arraycopy`）也非常快
2. **极少场景用 LinkedList** — 只有频繁在头部插入/删除时才考虑。典型场景：实现双端队列（Deque），但 JDK 已有 `ArrayDeque`，大多数情况下比 `LinkedList` 更好
3. **不要被 O(n) 迷惑** — LinkedList 的 O(1) 插入隐藏了遍历定位的成本。对于中间插入，两者都是 O(n)

### 面试中的回答

> "在日常开发中我几乎总是用 ArrayList，因为它的内存连续性好、缓存友好，而且 `System.arraycopy` 是 native 方法，拷贝速度很快。LinkedList 只在确实需要频繁头部操作的场景才会考虑，但即使这种情况下 ArrayDeque 往往更合适。"
