# 17 - AI 大模型

> AI 大模型是当前技术热点。面试中常问 RAG、幻觉、Function Calling、MCP 等概念，以及 AI 工具的实际使用经验。

---

## 1. 如何理解大模型？

大模型（LLM, Large Language Model）是通过海量文本数据训练的深度学习模型。核心特征是：

- **规模巨大** — 参数量从数十亿到数千亿不等
- **涌现能力** — 当模型规模超过某个临界值，会突然展现出训练时未显式教过的能力（如推理、翻译、编程）
- **上下文学习** — 通过 Prompt 中的示例就能完成新任务，无需微调（In-Context Learning）
- **思维链** — 在 Prompt 中加上 "Let's think step by step"，模型推理能力显著提升

**GPT 训练流程：**
```
Pre-training（海量数据预训练，学习通用知识）
    ↓
SFT（有监督微调，学习遵循指令）
    ↓
RLHF（基于人类反馈的强化学习，对齐人类偏好）
```

## 2. 大模型的幻觉（Hallucination）

**定义：** 模型生成的内容看上去合理但与事实不符。

**原因：**
1. 训练数据不完整或包含错误信息
2. 模型只学过训练数据中存在的知识
3. 生成方式是逐 token 概率采样，没有真正的"事实核查"机制

**缓解方法：**
1. **RAG（检索增强生成）** — 先检索相关文档，再基于文档生成回答
2. **限制回答范围** — Prompt 中明确 "如果不知道请说不知道"
3. **结构化输出** — 要求输出特定格式，减少自由发挥空间
4. **多轮验证** — 生成后让模型自我检查

---

## 3. RAG（检索增强生成）

### 核心流程

```
用户提问 → Query 向量化 → 向量相似度检索 → 召回相关文档
→ 拼接 Prompt（问题 + 召回文档）→ LLM 生成 → 返回答案
```

### 关键技术点

| 环节 | 关键点 |
|:---|:---|
| **文档切割** | 合适的分块大小（500-1000 tokens），重叠保留上下文连贯性 |
| **向量化模型** | text-embedding-3, bge-large-zh 等 |
| **向量数据库** | Milvus / Pinecone / pgvector / Weaviate |
| **检索优化** | 动态 TopK、相似度阈值、Query Rewrite、混合检索（向量 + 关键词） |
| **重排序（Rerank）** | 检索后用 Cross-Encoder 对结果重新排序 |

### RAG 的优势

- 回答基于真实文档，减少幻觉
- 知识可动态更新（不重新训练模型）
- 可溯源（可展示引用的文档段落）

## 4. Function Calling & MCP

### Function Calling

模型不直接执行函数，而是输出结构化的函数调用请求（函数名 + 参数 JSON），应用代码解析后执行并返回结果给模型。

```
用户: "北京今天天气怎么样？"
       ↓
模型输出: { "function": "get_weather", "arguments": { "city": "北京" } }
       ↓
应用调用天气 API，返回结果
       ↓
模型: "北京今天晴天，气温 25°C..."
```

### MCP（Model Context Protocol）

Anthropic 提出的标准化协议，让 AI 模型能安全地访问外部工具和数据源：

```
AI 应用（Host）←→ MCP Client ←→ MCP Server ←→ 外部资源
                                    ↓
                      本地文件 / 数据库 / API / 浏览器
```

**核心目标：** 标准化工具接入方式，不再需要每个 AI 应用单独对接每个工具（类似 USB 协议之于外设）。

## 6. 如何看待 DeepSeek？

**DeepSeek 的核心突破：**

1. **低成本高性能** — DeepSeek-V3 使用 MoE（Mixture of Experts）架构，在远低于 GPT-4 的训练成本下达到可比性能
2. **开源** — 模型权重开源，极大降低了 AI 应用的门槛
3. **推理能力强** — DeepSeek-R1 通过强化学习实现深度推理能力，展示了 "思考过程"（类似 o1）
4. **行业影响** — 证明了 "大力出奇迹" 不是唯一路线，工程优化同样重要

**面试中可以提到的观点：**

> "DeepSeek 的出现对整个 AI 行业产生了深远影响。它证明了通过技术创新可以在较低成本下实现顶级性能，打破了 '训练算力 = 模型能力' 的固有认知。特别是 DeepSeek-R1 的推理能力，对需要复杂推理的企业应用场景（如代码审查、法律分析、医疗诊断）有很大价值。作为开发者，我更关注它开源带来的生态红利 — 可以在本地部署、微调，构建更可控的 AI 应用。"
