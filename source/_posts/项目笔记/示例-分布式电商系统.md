## 1. 项目概述

### 1.1 项目背景

为了深入学习分布式系统开发，实现一个完整的电商系统，涵盖用户管理、商品管理、购物车、订单系统、支付系统等核心模块。

### 1.2 核心功能

- **用户服务**：注册、登录、JWT认证、用户信息管理
- **商品服务**：商品列表、详情、分类、库存管理
- **购物车服务**：添加商品、修改数量、选中结算
- **订单服务**：创建订单、订单状态管理、订单查询
- **支付服务**：集成支付宝沙箱、支付回调
- **消息服务**：订单超时取消、库存回滚

## 2. 技术亮点

### 2.1 亮点 1：Redis分布式锁防止超卖

**问题场景：**

高并发场景下，多个用户同时购买同一件库存为1的商品，如果不加控制会出现超卖。

**解决方案：**

使用Redis分布式锁，基于Redisson实现可重入锁。

**关键代码：**

```java
@Service
public class StockServiceImpl implements StockService {
    
    @Autowired
    private RedissonClient redissonClient;
    
    @Autowired
    private StringRedisTemplate redisTemplate;
    
    private static final String STOCK_KEY = "stock:%d";
    private static final String LOCK_KEY = "lock:stock:%d";
    
    /**
     * 扣减库存
     */
    @Override
    public boolean deductStock(Long productId, Integer quantity) {
        String lockKey = String.format(LOCK_KEY, productId);
        RLock lock = redissonClient.getLock(lockKey);
        
        try {
            // 尝试获取锁，等待3秒，锁定5秒（看门狗自动续期）
            boolean locked = lock.tryLock(3, 5, TimeUnit.SECONDS);
            if (!locked) {
                log.warn("获取库存锁失败，productId={}", productId);
                return false;
            }
            
            // 获取当前库存
            String stockKey = String.format(STOCK_KEY, productId);
            String stockStr = redisTemplate.opsForValue().get(stockKey);
            
            if (stockStr == null) {
                // 从数据库加载库存到Redis
                loadStockFromDB(productId);
                stockStr = redisTemplate.opsForValue().get(stockKey);
            }
            
            int stock = Integer.parseInt(stockStr);
            
            // 检查库存
            if (stock < quantity) {
                log.warn("库存不足，productId={}, stock={}, quantity={}", 
                    productId, stock, quantity);
                return false;
            }
            
            // 扣减库存
            redisTemplate.opsForValue().decrement(stockKey, quantity);
            
            // 发送库存变更消息，异步同步到数据库
            stockChangeProducer.send(new StockChangeMessage(productId, -quantity));
            
            return true;
            
        } catch (InterruptedException e) {
            Thread.currentThread().interrupt();
            log.error("扣减库存异常", e);
            return false;
        } finally {
            // 释放锁（检查是否当前线程持有）
            if (lock.isHeldByCurrentThread()) {
                lock.unlock();
            }
        }
    }
}
```

**效果/收益：**

- 压测结果：单机QPS从500提升到3000
- 保证库存一致性，未出现超卖

### 2.2 亮点 2：订单超时自动取消

**问题场景：**

用户下单后需要在30分钟内完成支付，超时未支付订单自动取消，库存回滚。

**解决方案：**

使用RabbitMQ延迟队列实现订单超时处理。

**关键代码：**

```java
@Configuration
public class RabbitMQConfig {
    
    // 订单交换机
    @Bean
    public DirectExchange orderExchange() {
        return new DirectExchange("order.exchange");
    }
    
    // 延迟队列（死信队列）
    @Bean
    public Queue orderDelayQueue() {
        Map<String, Object> args = new HashMap<>();
        // 消息过期后发送到 order.cancel.queue
        args.put("x-dead-letter-exchange", "order.exchange");
        args.put("x-dead-letter-routing-key", "order.cancel");
        // 消息过期时间：30分钟
        args.put("x-message-ttl", 30 * 60 * 1000);
        return QueueBuilder.durable("order.delay.queue").withArguments(args).build();
    }
    
    // 取消订单队列
    @Bean
    public Queue orderCancelQueue() {
        return QueueBuilder.durable("order.cancel.queue").build();
    }
    
    @Bean
    public Binding orderDelayBinding() {
        return BindingBuilder.bind(orderDelayQueue())
            .to(orderExchange()).with("order.create");
    }
    
    @Bean
    public Binding orderCancelBinding() {
        return BindingBuilder.bind(orderCancelQueue())
            .to(orderExchange()).with("order.cancel");
    }
}

@Service
public class OrderServiceImpl implements OrderService {
    
    @Autowired
    private RabbitTemplate rabbitTemplate;
    
    /**
     * 创建订单
     */
    @Override
    @Transactional
    public Order createOrder(CreateOrderDTO dto) {
        // 1. 扣减库存
        boolean success = stockService.deductStock(dto.getProductId(), dto.getQuantity());
        if (!success) {
            throw new BusinessException("库存不足");
        }
        
        // 2. 创建订单
        Order order = new Order();
        order.setOrderNo(generateOrderNo());
        order.setUserId(dto.getUserId());
        order.setStatus(OrderStatus.UNPAID);
        // ... 设置其他字段
        orderMapper.insert(order);
        
        // 3. 发送延迟消息
        OrderDelayMessage message = new OrderDelayMessage();
        message.setOrderId(order.getId());
        message.setOrderNo(order.getOrderNo());
        message.setCreateTime(LocalDateTime.now());
        
        rabbitTemplate.convertAndSend(
            "order.exchange", 
            "order.create", 
            message
        );
        
        return order;
    }
}

@Component
public class OrderCancelListener {
    
    @Autowired
    private OrderService orderService;
    
    @Autowired
    private StockService stockService;
    
    @RabbitListener(queues = "order.cancel.queue")
    public void handleOrderCancel(OrderDelayMessage message) {
        log.info("收到订单取消消息，orderId={}", message.getOrderId());
        
        // 查询订单状态
        Order order = orderService.getById(message.getOrderId());
        
        // 只有未支付订单才取消
        if (order != null && order.getStatus() == OrderStatus.UNPAID) {
            // 取消订单
            orderService.cancelOrder(order.getId());
            
            // 回滚库存
            stockService.rollbackStock(order.getProductId(), order.getQuantity());
            
            log.info("订单已自动取消，orderId={}", message.getOrderId());
        }
    }
}
```

**时序图：**

```
用户        订单服务        延迟队列        取消服务        库存服务
 |             |               |               |               |
 |---下单---->|               |               |               |
 |             |---扣减库存------------------------------------>|
 |             |<--成功--------|               |               |
 |             |---发送延迟消息--------------->|               |
 |<--返回-----|               |               |               |
 |             |               |   30分钟后    |               |
 |             |               |---转发消息-->|
 |             |               |               |---查询订单状态-->|
 |             |               |               |<--未支付-------|
 |             |               |               |---取消订单---->|
 |             |               |               |---回滚库存---->|
 |             |               |               |               |
```

## 3. 技术难点与解决方案

### 3.1 难点 1：Redis和MySQL数据一致性

**挑战：**

缓存和数据库可能出现不一致，如更新数据库后删除缓存失败。

**尝试方案：**

- 方案 A：先删缓存，再更新数据库（并发读会导致不一致）
- 方案 B：先更新数据库，再删缓存（删除失败会不一致）

**最终方案：**

采用"先更新数据库，再删缓存"+"延迟双删"+"消息补偿"

```java
@Service
public class ProductServiceImpl implements ProductService {
    
    @Autowired
    private ProductMapper productMapper;
    
    @Autowired
    private StringRedisTemplate redisTemplate;
    
    @Autowired
    private DelayMessageProducer delayMessageProducer;
    
    @Override
    @Transactional
    public void updateProduct(Product product) {
        Long productId = product.getId();
        String cacheKey = "product:" + productId;
        
        // 1. 删除缓存
        redisTemplate.delete(cacheKey);
        
        // 2. 更新数据库
        productMapper.updateById(product);
        
        // 3. 发送延迟消息，延迟删除缓存（防止并发读导致脏数据）
        delayMessageProducer.send(new CacheDeleteMessage(cacheKey), 500);
    }
}
```

**踩坑记录：**

- 延迟时间设置太短（100ms）还是有并发问题，改为500ms后解决
- 延迟队列使用Redis的zset实现，比RabbitMQ更简单

**Q1: 请介绍一下这个项目**

1. 项目背景：学习分布式系统
2. 核心功能：电商完整流程
3. 技术亮点：分布式锁、延迟队列、分布式事务
4. 个人收获：对微服务架构有了深入理解

**回答模板：**

> 这个项目是一个基于Spring Cloud的分布式电商系统，主要为了深入学习微服务架构。项目包含用户、商品、订单、支付等核心模块，使用Nacos做服务注册发现，Gateway做统一网关，Seata处理分布式事务。我在项目中主要解决了高并发下的库存扣减问题，使用Redis分布式锁防止超卖，以及使用RabbitMQ延迟队列实现订单超时自动取消。

---

**Q2: 项目的难点是什么？如何解决的？**

**参考回答：**

最大的难点是防止超卖问题。一开始我们直接用数据库行锁，但性能太差，QPS只有500。后来改为Redis预扣库存+异步同步数据库，用Redisson实现分布式锁，单机QPS提升到3000。另外为了保证数据一致性，我们使用了延迟双删策略处理缓存不一致问题。

---

**Q3: 如果重新做这个项目，你会怎么改进？**

**思考方向：**
- **分库分表**：目前单库压力还大，可以按用户ID或订单ID分片
- **多级缓存**：本地缓存+Redis缓存，减少网络开销
- **服务网格**：使用Istio替代部分Spring Cloud组件

---

### 5.2 技术细节类

**Q4: Redisson分布式锁的实现原理？**

**答案：**

Redisson使用Lua脚本保证加锁和解锁的原子性，使用看门狗机制自动续期。加锁时设置过期时间防止死锁，同时启动定时任务，在锁即将过期时续期。

---

### 5.3 设计类

**Q5: 项目的架构设计是怎样的？为什么这么设计？**

**回答：**

采用微服务架构，按业务领域拆分服务。这样每个服务职责单一，便于独立开发和部署。服务间通过OpenFeign调用，使用Nacos做服务发现，Gateway做统一入口。这样设计的好处是扩展性好，某个服务压力大时可以单独扩容。
