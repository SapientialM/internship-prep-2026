# 09 - 操作系统

> 操作系统基础知识在面试中主要考察进程线程、内存管理、IO 模型，和并发编程强相关。

---

## 1. 进程和线程

| 维度 | 进程 | 线程 |
|:---|:---|:---|
| 定义 | 资源分配的基本单位 | CPU 调度的基本单位 |
| 内存 | 各进程独立地址空间 | 同进程内线程共享地址空间 |
| 通信 | IPC（管道、消息队列、共享内存、Socket） | 共享内存（需要注意同步） |
| 切换开销 | 大（切换页表、TLB 刷新等） | 小（同进程内只需切换寄存器和栈） |
| 崩溃影响 | 不会影响其他进程 | 可能导致整个进程崩溃 |

### 协程（Coroutine）

一种用户态的轻量级线程，调度由程序自身控制而非内核。Go 的 Goroutine、Kotlin 的协程、Java 的虚拟线程（Virtual Threads, JDK 21）都是协程的实现。

**Virtual Threads 的优势：** 线程数可以达到百万级（传统 OS 线程数千个就会 OOM），IO 操作自动 yield 而不是阻塞 OS 线程。

## 2. 进程间通信（IPC）

| 方式 | 说明 |
|:---|:---|
| 管道（Pipe） | 半双工，父子进程间通信 |
| 命名管道（FIFO） | 任意进程间通信 |
| 消息队列 | 异步消息传递，支持多对多 |
| 共享内存 | 最快的方式，但需要同步机制 |
| 信号量（Semaphore） | 用于进程间同步，不是数据传递 |
| Socket | 支持跨网络通信 |

## 3. 死锁

### 四个必要条件

1. **互斥** — 资源只能被一个进程占有
2. **占有并等待** — 占有一个资源的同时等待另一个
3. **不可抢占** — 资源不能被强制释放
4. **循环等待** — 形成一个循环等待链

### 避免死锁的方式

1. **破坏循环等待** — 统一获取锁的顺序（如按资源 ID 升序加锁）
2. **破坏占有并等待** — 一次性申请所有资源
3. **引入超时** — `tryLock(timeout)` 获取不到就释放已有资源
4. **死锁检测** — 定期检测是否有死锁，有则强制释放

## 4. 内存管理

### 虚拟内存

每个进程有自己的虚拟地址空间，通过 MMU（内存管理单元）将虚拟地址映射到物理地址。

**好处：**
1. 进程隔离（一个进程无法访问另一个进程的内存）
2. 物理内存利用率更高（按需分页 + 页面置换）
3. 简化程序加载（可加载到非连续物理内存中）

### 缺页中断

当虚拟地址对应的物理页面不在内存中时，触发缺页中断：
1. 检查访问是否合法
2. 从磁盘加载页面到内存
3. 更新页表映射
4. 重新执行引起缺页的指令

## 5. IO 模型

| 模型 | 说明 | 特点 |
|:---|:---|:---|
| 阻塞 IO | 应用进程阻塞直到数据准备好 | 简单但效率低 |
| 非阻塞 IO | 轮询检查数据是否准备好 | 消耗 CPU |
| IO 多路复用 | select/poll/epoll 监听多个 fd | 单线程管理多个连接 |
| 信号驱动 IO | 内核发 SIGIO 信号通知就绪 | 不常用 |
| 异步 IO | 内核完成所有操作后通知 | 真正的异步（Java NIO2 AIO） |

### select / poll / epoll 区别

| 特性 | select | poll | epoll |
|:---|:---|:---|:---|
| 数据结构 | 固定大小数组 | 链表 | 红黑树 + 就绪链表 |
| fd 数量限制 | 1024（可改编译） | 无限制 | 无限制 |
| 扫描方式 | 遍历所有 fd (O(n)) | 遍历所有 fd (O(n)) | 只返回就绪 fd (O(1)) |
| 触发方式 | 水平触发 | 水平触发 | 支持 ET/LT |

## 6. 用户态和内核态

**切换场景：** 系统调用（read/write）、中断、异常。

**切换开销：**
1. 保存用户态寄存器、PC、栈指针
2. 切换到内核栈
3. 修改 CPU 特权级
4. 完成后恢复用户态上下文

**零拷贝：** 避免 CPU 将数据从内核空间拷贝到用户空间。Kafka 使用 `sendfile()` 系统调用，数据从 Page Cache 直接发送到网卡，不经过用户态。
